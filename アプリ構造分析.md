# 📋 アプリ概要：「Dormire（ねむりや）」バックエンドAPI

睡眠寝具メーカー「**ねむりや（Dormire）**」が提供する、**VP計測（Visual Posture計測）→ AIレコメンド → 注文**を一気通貫で扱う **Django REST Framework製のBtoB/BtoC寝具オーダーメイドシステム**のバックエンドです。iOSアプリ（Dormire）と連携して動作します。

## 🏗 技術スタック

| 領域 | 内容 |
|---|---|
| 言語/FW | Python 3.10 / Django 4.2 / DRF 3.16 |
| 認証 | django-allauth + dj-rest-auth（トークン認証） |
| DB | PostgreSQL 14 |
| ストレージ | S3互換（本番: さくらオブジェクトストレージ、開発: MinIO） |
| 画像 | Pillow + Base64アップロード |
| メール | AWS SES (SMTP) |
| デプロイ | Docker / Gunicorn (systemd) / uWSGI |
| バックアップ | pg_dump → S3（7日ローカル、30日S3保持） |

### Djangoプロジェクト（[ソースコード/](ソースコード/)）

#### 1. [config/](ソースコード/config/) — プロジェクト設定
- [settings.py](ソースコード/config/settings.py) — PostgreSQL、S3、CORS、DRF（camelCase変換）、トークン認証、SES設定
- [urls.py](ソースコード/config/urls.py) — ルーティング（admin / auth / recommends / measure / orders / health）
- [s3boto3.py](ソースコード/config/s3boto3.py) — S3カスタムストレージ

#### 2. [users/](ソースコード/users/) — ユーザー管理
- [models.py](ソースコード/users/models.py) — カスタムUserモデル（UUID主キー、email認証）
  - 権限タイプ：`admin` / `salon`（サロン） / `furusato_tax`（ふるさと納税） / `crowd_funding`（クラウドファンディング）
- [permissions.py](ソースコード/users/permissions.py) — DRFカスタムパーミッション
- [admin.py](ソースコード/users/admin.py) — CSV/Excel一括インポート時にパスワード自動生成＆メール送信
- [management/commands/custom_createsuperuser.py](ソースコード/users/management/commands/custom_createsuperuser.py) — 初期管理者作成

#### 3. [auth/](ソースコード/auth/) — 認証エンドポイント
- `POST /signin/` — dj-rest-authのLoginViewでトークン取得

#### 4. [measure/](ソースコード/measure/) — VP計測（整体師モード）
身体を計測して得られた約25項目の計測値（腕長・脚長・肩角度・骨盤角度・首幅・頭幅・姿勢タイプなど）を保存。

- [models.py](ソースコード/measure/models.py) に2モデル：
  - **`Measure`** — オーダー連携用のVP計測結果
  - **`MeasureOnly`** — 整体師モード用（顧客情報付き、単体保存）
- [views.py](ソースコード/measure/views.py) — `POST /measure/` で `MeasureOnly` を保存し、**ユーザーへ解析結果メール送信**
- 権限：`IsAdmin | IsSalonUser`（整体師サロン店員のみ）
- 画像は base64 で受け取り、S3 の `vp-images/` 配下に保存

#### 5. [recommends/](ソースコード/recommends/) — 寝具レコメンドエンジン（アプリの頭脳）
**アプリの最重要モジュール**。ヒアリング30問 + VP計測データ + BMIから、**枕とマットレスを個別最適化**して返す。

- [models.py](ソースコード/recommends/models.py) の主要モデル：
  - `Hearing` — 30問の問診結果
  - `MatBase` — 性別/BMI/身長/腰痛/肩痛/姿勢タイプごとのマットレスベーステンプレート（インデックス付き）
  - `PillowBase` — 素材別の枕ベース高さ
  - `MatRec` / `PillowRec` — 算出結果（6分割した各ポジションの高さ・硬さ、サポートパッド位置×4種）
- [serializers.py](ソースコード/recommends/serializers.py) — `HearingSerializer.create()` がコア：
  1. VP計測データ保存
  2. BMI算出
  3. `generate_recommend_mat_by_data()` — 性別・身長・BMI・腰痛有無・姿勢タイプで `MatBase` を検索し、サポートパッド配置（`generate_recommend_support_pad_place_by_data()`）を決定
  4. `generate_recommend_pillow_by_data()` — 素材選択・身長/BMI・姿勢から6ポジションの枕高さを計算し、シート厚みを決定
- `POST /recommends/` で一括処理し、`mat_rec` と `pillow_rec` を返却
- ⚠ README 注記：現状「**身長159cm未満・姿勢タイプN**」しか実装されていない

#### 6. [orders/](ソースコード/orders/) — 注文処理
レコメンド結果を基に実注文を登録。

- [models.py](ソースコード/orders/models.py) のモデル：
  - `Customer` — 配送先情報
  - `PillowOrder` — 枕注文（素材・カバー素材・カバー色×3、色ごとの枚数管理）
  - `MatOrder` — マットレス注文（サイズ・厚み・カバー、**2枚のMatRecを合成可能**＝夫婦で異なる推奨）
  - `Order` — 統合注文（枕×最大2個 + マットレス + ヒアリング最大2名分）
- [serializers.py](ソースコード/orders/serializers.py) — `@transaction.atomic` で顧客・枕・マットレスをまとめて作成。1人目と2人目のヒアリング整合性をチェック
- [views.py](ソースコード/orders/views.py) — 作成後に顧客へ**注文確認メール**を自動送信

#### 7. [templates/](ソースコード/templates/) — メールテンプレート
- `measure_mail.html` — 計測結果通知
- `order_mail.html` — 注文完了通知
- `user_registration_mail.txt` — 新規ユーザー登録通知

- [sql/init.sql](ソースコード/sql/init.sql) — DB初期化SQL

## 🔁 業務フロー（全体の動き）

```
┌─────────────┐   ①サロン店員がログイン          ┌──────────┐
│ iOS アプリ  │ ───────────────────────────────▶│  /signin/ │
│ (Dormire)   │ ◀── トークン ────────────────── └──────────┘
└─────────────┘
       │
       │ ②顧客をカメラで撮影→骨格認識→計測値生成
       │
       ├─ Aパターン（整体師モード・計測のみ）
       │    POST /measure/
       │    └─ MeasureOnlyに保存 → 顧客にメール送信
       │
       └─ Bパターン（ヒアリング込み・寝具販売）
            POST /recommends/  ※30問問診＋VP計測＋属性
            └─ BMI算出 → MatBase検索 → サポートパッド計算
                       → 枕ポジション×6算出 → シート厚決定
            ◀── mat_rec_id / pillow_rec_id ──

            POST /orders/      ※顧客情報＋枕×1-2＋マットレス
            └─ Customer/PillowOrder/MatOrder/Order作成
               → 顧客に注文確認メール送信
```

## 🎯 ドメインサマリー

このシステムは **「姿勢計測に基づく完全オーダーメイド寝具（枕・マットレス）」** を販売するビジネスのためのAPIです。

- **計測**：iOSで撮影した身体画像から25項目の特徴量を取得
- **レコメンド**：問診30問 + 計測 + BMI から、6分割された枕・マットレスの各部位の高さ/硬さをアルゴリズムで決定
- **販売**：整体サロン（`IsSalonUser`）やふるさと納税・クラウドファンディング経由の顧客を区別して権限管理
- **業務連携**：結果をメール送信し、Django管理画面でスタッフが注文・顧客データを閲覧／エクスポート可能

つまり、**「ねむりや（Dormire）」ブランドの整体サロン＆ECサービスが、iOSアプリで顧客の姿勢を測定し、その人専用の枕とマットレスを設計して販売するためのバックエンド**です。
