# 🛏 このアプリって何？ 〜素人向けざっくり解説〜

## ひとことで言うと

**「スマホで身体を撮影するだけで、あなた専用の枕とマットレスを作ってくれる寝具オーダーメイドサービス」**

の裏側で動いているシステムです。ブランド名は「**ねむりや（Dormire／ドルミーレ）**」といいます。

---

## 🎬 お客さんの体験ストーリー

整体サロンに行ったと想像してください。

### 📍 Step 1：お店に行く
あなたが整体サロンに来店します。店員さんが **iPadやiPhoneのアプリ（Dormire）** を取り出します。

### 📍 Step 2：カメラで撮影
店員さんがあなたを**カメラで撮影**。すると、アプリが自動で…
- 腕の長さ
- 足の長さ
- 肩の角度
- 骨盤の傾き
- 首の幅
- 背中の角度
- 姿勢のタイプ

など、**25項目もの身体データ**を読み取ります。まるで健康診断のような感じです。

### 📍 Step 3：30個の質問に答える
次に、店員さんからタブレットで質問されます。
- 「腰痛はありますか？」
- 「肩こりはありますか？」
- 「普段どんな姿勢で寝ていますか？」
- 「好きな枕の素材は？（パイプ？炭入り？）」
- 「マットレスの好みの硬さは？」

…などなど**全30問**。身長・体重・年齢・性別も入力します。

### 📍 Step 4：AIが考える（ここが一番スゴい！）
アプリが裏側でこんな計算をします：

```
あなたの身体データ + 30問の答え + BMI
         ↓
🧠 独自アルゴリズムがグルグル計算
         ↓
「このお客さんには…
  ・枕は6つのエリアに分けて、この高さで！
  ・マットレスはこの硬さで、腰の部分には
   サポートパッドを入れよう！」
```

ポイントは、**枕もマットレスも「ひとつの塊」じゃなく、「6つのエリアに分けて」それぞれ最適な高さ・硬さにする**こと。頭・首・肩・腰・膝・足首…と、身体のカーブにピッタリ合うように設計されます。

### 📍 Step 5：注文する
店員さんが「おすすめはこれですよ〜」と画面を見せてくれます。

- 枕の素材（ソフトパイプ／エラストマーパイプ／備長炭パイプ）
- 枕カバーの色（グレー／ベージュ／ピンク）
- マットレスのサイズ・厚み・カバー色

を選んで **注文確定**！

ちなみに **夫婦で2人分をまとめて注文できる**ようにもなっています（「1人目」「2人目」と分けて管理できる）。

### 📍 Step 6：メールが届く
注文が終わると、登録したメールアドレスに **「ご注文ありがとうございます」メール**が自動で届きます。あとは商品が家に届くのを待つだけ。

## 💼 誰が使うの？

このシステムには **4種類のユーザー**がいます：

1. 👑 **管理者（admin）** — 会社のスタッフ。すべての注文やお客様情報を見られる
2. 💆 **サロン店員（salon）** — 整体サロンで働く人。お客さんを計測して注文できる
3. 🎁 **ふるさと納税ユーザー** — ふるさと納税の返礼品として商品を受け取る人
4. 💰 **クラウドファンディング支援者** — クラファンで先行購入した人

このように、**「どの経路でお客さんが来たか」で区別**して管理できるようになっています。

## 🔧 エンジニアっぽく言うと…

素人の人は読み飛ばしてOKですが、技術的には：

- **Python**という言語で作られた**Django**というフレームワーク
- データは**PostgreSQL**という大型データベースに保存
- 写真は**S3**（クラウドのファイル保管庫）に保存
- **iPhoneアプリ**と**インターネット経由で通信**する仕組み（REST API）
- 毎日自動で**データのバックアップ**を取っている

---

## 🎯 つまりこのシステムは…

> 💡 **「ひとりひとりの身体に合わせた、世界に1つだけの枕とマットレスを作るためのコンピューターの仕組み」**

普通の寝具店だと「S・M・L」みたいな既製品しかありませんよね。でもこのサービスは、**AIが身体データを読み取って「あなたの身体のカーブにピッタリ合う寝具」を自動設計**してくれる、ちょっと未来的なオーダーメイド寝具屋さんなんです。

整体師さんが「この人には硬めのマットがいいな…」と経験で判断していたことを、**コンピューターが数値で正確に計算してくれる**、そんなシステムですね 😊

## 1. 個人情報入力のスリム化

### 現状
整体師モード（[measure/](ソースコード/measure/)）では、お客さんから以下の情報を入力してもらっています：

- 氏名（漢字）
- 氏名（カナ）
- 電話番号
- メールアドレス
- 郵便番号
- 住所
- 建物名

### 改修内容
**個人情報の入手はスリム化したいので、氏名だけにする。**

- ✅ 残す項目：**氏名のみ**
- ❌ 削除する項目：カナ・電話番号・メールアドレス・郵便番号・住所・建物名

### 影響範囲（改修時に触る場所）
- [measure/models.py](ソースコード/measure/models.py) — `MeasureOnly` モデルから不要フィールドを削除
- [measure/serializers.py](ソースコード/measure/serializers.py) — `MeasureOnlySerializer` の対応
- [measure/migrations/](ソースコード/measure/migrations/) — マイグレーションファイルの追加
- [measure/views.py](ソースコード/measure/views.py) — メール送信処理（`send_email_on_measure`）は `user.email` を使っているので影響なし
- Django管理画面（admin.py）— 表示項目の見直し

## 2. ヒアリング質問項目の削減（30問 → 10問）

### 現状
[recommends/models.py](ソースコード/recommends/models.py) の `Hearing` モデルには **質問1〜30の30項目**があり、`POST /recommends/` で全て入力させています。

しかし [recommends/serializers.py](ソースコード/recommends/serializers.py) のレコメンドアルゴリズムを解析した結果、**実際に計算で使われているのは12問のみ**で、残り18問は保存されているだけで未使用でした。

### 改修内容
**ヒアリング質問を10問に絞り、ユーザーの入力負担を軽減する。**

以下の **10問だけを残す**：

| № | 質問 | 回答型 | 用途 |
|---|---|---|---|
| 1 | 首が痛いことがある | 数値 | 枕の左右位置調整 |
| 2 | 肩が痛いことがある | 数値 | マットレスの肩痛フラグ・枕調整 |
| 3 | 腰が痛いことがある | 数値 | マットレスの腰痛フラグ |
| 4 | いびきがある | 文字列 | 枕の高さ微調整 |
| 5 | 反り腰がある | 文字列 | サポートパッド配置 |
| 6 | 現在使用マットレスの柔らかさ | 文字列 | 枕の高さ微調整 |
| 7 | 好みのマットレスの柔らかさ | 文字列 | **マットレスの硬さ決定** |
| 8 | 好みの枕の高さ | 文字列 | **枕の高さ決定** |
| 9 | 首元はしっかり支えたい | 文字列 | 枕の高さ調整 |
| 10 | 好みの枕の素材 | 文字列 | **枕の素材決定（必須）** |

### 削除する質問（全20問）

**睡眠習慣系（8問）** — 記録のみで未使用
- Q1 よく眠る姿勢 / Q2 よく眠れていますか / Q3 布団に入る時間 / Q4 起きる時間
- Q5 寝入りまでの時間 / Q6 平日の平均睡眠時間 / Q7 途中で起きますか / Q8 夜中にトイレに行く

**体調系（3問）** — 記録のみで未使用
- Q9 昼間に眠気を感じる / Q10 頭痛 / Q17 冷え性がある

**痛みの左右（2問）** — 左右非対称な微調整にのみ使用 → **今回の改修で一律両側ケアに変更**
- Q12 首のどちらが痛いか / Q14 肩のどちらが痛いか

**体質・姿勢（2問）** — 記録のみで未使用
- Q19 睡眠薬について / Q21 猫背がある

**現状把握（3問）** — 記録のみで未使用
- Q22 現在の寝方 / Q25 現在のまくらの使用年数 / Q26 現在のマットレスの使用年数

**その他（2問）**
- Q30 自由記入欄 （※残したい場合は再考）

### 注意点・影響
- **痛みの左右判定（Q12/Q14）を削除するため、枕の左右非対称な微調整ができなくなる**。今後は「片側痛い人」にも両側対応の枕を出す仕様になる。
- 削除する質問も**過去データとしてDBには残す**ので、マイグレーションは **新規作成時に必須ではなくする（null=True に変更）** 方針を推奨。
- 逆に完全削除する場合は、過去の `Hearing` レコードが全て消えるリスクがあるため慎重に。

### 影響範囲（改修時に触る場所）
- [recommends/models.py](ソースコード/recommends/models.py) — `Hearing` モデルの不要フィールドを `null=True` 化 or 削除
- [recommends/serializers.py](ソースコード/recommends/serializers.py) — `HearingSerializer` から `hearing1`〜`hearing30` のうち10問以外を削除。`Hearing1Serializer`〜`Hearing30Serializer` も該当分のみ残す
- [recommends/migrations/](ソースコード/recommends/migrations/) — マイグレーションファイルの追加
- [swagger/openapi.yaml](ソースコード/swagger/openapi.yaml) — API仕様書の更新（`RecommendRequest` と `Hearing1`〜`Hearing30` スキーマの整理）
- [orders/admin.py](ソースコード/orders/admin.py) — 管理画面の表示項目見直し
- **iOSアプリ側**も同時に改修が必要（質問画面のUI変更）

# 📸 現在の計測フロー詳細

## 🆚 ノーマルモード vs 整体師モード

|  | 🛒 ノーマルモード | 💆 整体師モード |
|---|---|---|
| **誰が使う？** | 誰でも（一般ユーザー・サロン店員・ふるさと納税・クラファン支援者） | **サロン店員＆管理者のみ**（`IsSalonUser` or `IsAdmin`） |
| **目的** | **寝具を購入する**ため | **計測結果レポートを顧客に渡す**ため |
| **使うAPI** | `POST /recommends/` → `POST /orders/` | `POST /measure/` |
| **保存先テーブル** | `Measure` | `MeasureOnly` |
| **ヒアリング質問** | ✅ 30問答える | ❌ 無し |
| **レコメンド計算** | ✅ 枕・マットレスを自動設計 | ❌ 無し |
| **注文に繋がる？** | ✅ 購入可能 | ❌ 計測のみで終了 |
| **顧客情報の入力** | ❌ 不要（ログインユーザー情報を使用） | ✅ 氏名・住所など毎回入力 |
| **メール送信** | 注文確認メール → 顧客へ | 解析結果メール → 店員へ |

### ひとことで言うと
- **ノーマルモード** = 「**計測→AI診断→購入**」の1本のパイプライン（販売あり）
- **整体師モード** = 「**計測→結果レポート**」だけの単発サービス（販売なし）

### コード上の根拠
- [measure/models.py:70-73](ソースコード/measure/models.py) — `MeasureOnly` の `verbose_name = "VP計測結果（整体師モード）"`
- [measure/models.py:18-21](ソースコード/measure/models.py) — `Measure` の `verbose_name = "VP計測結果（オーダー用）"`
- [measure/views.py:17](ソースコード/measure/views.py) — `permission_classes = (IsAuthenticated, IsAdmin | IsSalonUser)`
- [swagger/openapi.yaml:106](ソースコード/swagger/openapi.yaml) — `/measure/` の説明 "整体師モードによってVisionPoseでの測定のみを行う"

## 📸 撮影の位置づけについて

### ⚠️ 重要ポイント
**撮影・骨格認識・25項目の計測は、すべて iOSアプリ側で完結**しています。
バックエンドは **「既に計算された数値と画像」を受け取るだけ** です。

```
┌─────────────────────────────────┐
│ 📱 iOSアプリ側で完結             │
│                                 │
│  📸 撮影（2枚）                  │
│     ↓                           │
│  🦴 骨格認識（Apple Vision等）   │
│     ↓                           │
│  📐 25項目を自動算出             │
│     ↓                           │
│  🔢 JSON化 + Base64エンコード    │
└────────────┬────────────────────┘
             │
             │ HTTP POST
             │ （画像 + 数値）
             ▼
┌─────────────────────────────────┐
│ 🖥 バックエンド（Django）         │
│                                 │
│  💾 データを保存するだけ         │
│  ☁️ 画像をS3にアップロード        │
│  📧 メール送信                   │
│  🤖 （ノーマル時）レコメンド計算 │
└─────────────────────────────────┘
```

### 改修時の切り分け
- **撮影のUI・タイミング変更** → iOSアプリ側の改修
- **数値の保存・計算処理の変更** → バックエンド側（Django）の改修

## 📥 リクエストデータ詳細

### ノーマルモード（`POST /recommends/` → `POST /orders/`）

**/recommends/ で送るデータ:**
- 属性情報：`sex`, `age`, `height`, `weight`, `both`（夫婦同時購入フラグ）
- ヒアリング：`hearing1`〜`hearing30`（各 `{question, answer}`）
