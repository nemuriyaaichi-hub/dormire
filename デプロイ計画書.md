# 🚀 Dormire スマホブラウザ化 & Netlify デプロイ計画書

作成日: 2026-04-08

---

## 📋 目的

現状の iOS ネイティブアプリ（Swift + Apple Vision）を、**スマートフォンのブラウザから使える Web アプリ**に置き換え、**Netlify にデプロイ**して開通させる。

---

## 🎯 前提条件（ユーザー指定）

- ✅ **Netlify は必須**（他のホスティング選択不可）
- ✅ 対象デバイスは **スマートフォンのブラウザ**（iOS Safari / Android Chrome）
- ✅ **ログイン機能は廃止**（改修 #3 で決定済み）
- ✅ 必要なら **Python コードを書き換える**
- ✅ カメラで撮影 → 骨格認識 → 20項目算出 → レコメンド → 注文 の一連のフローを維持

---

## ⚠️ 技術制約（最重要）

Netlify で動かせるもの・動かせないものは以下の通り:

| カテゴリ | Netlify で動く？ |
|---|---|
| 静的ファイル (HTML/CSS/JS) | ✅ 動く |
| Netlify Functions (**Node.js / TypeScript / Go** のみ) | ✅ 動く |
| Netlify Edge Functions (Deno) | ✅ 動く |
| **Python / Django 常駐サーバー (uWSGI)** | ❌ **動かない** |
| **PostgreSQL データベース** | ❌ 動かない |
| **S3 互換ストレージ** | ❌ 動かない（Netlify Blobs で代替可） |
| SMTP / AWS SES | ❌ 動かない（外部サービス利用） |

### 結論

**「Netlify に全部乗せる」 = Django を捨てて JAMstack に作り直す** しか方法はない。

既存 Python コード（[recommends/serializers.py](ソースコード/recommends/serializers.py) の 1000行のアルゴリズム等）は**ロジックとして移植**するが、**フレームワーク（Django/DRF）は使えない**。

## 🔧 技術スタック詳細

### フロントエンド: Next.js 15

**選定理由**
- Netlify 公式サポート（`@netlify/plugin-nextjs` 有）
- App Router で API Routes と静的ページを統合
- PWA 化が容易（`next-pwa`）
- スマホブラウザで快適に動作

**PWA 化**
- `manifest.json` でホーム画面追加可能
- Service Worker でオフラインキャッシュ
- `apple-mobile-web-app-capable` で iOS Safari でも全画面化

### 骨格認識: MediaPipe Pose

**選定理由**
- Google 製、WebAssembly で高速
- iOS Safari / Android Chrome で動作実績あり
- 33点のランドマーク検出 → 20項目の算出に十分
- TensorFlow.js MoveNet より精度が高い（全身検出）

**移植する20項目のリスト** ([measure/models.py:31-52](ソースコード/measure/models.py#L31-L52))

| 項目 | MediaPipe ランドマークから算出方法 |
|---|---|
| right_arm_length | 右肩 (12) → 右肘 (14) → 右手首 (16) の距離和 |
| left_arm_length | 左肩 (11) → 左肘 (13) → 左手首 (15) の距離和 |
| right_leg_length | 右腰 (24) → 右膝 (26) → 右足首 (28) の距離和 |
| left_leg_length | 左腰 (23) → 左膝 (25) → 左足首 (27) の距離和 |
| knee_ankle | 膝 → 足首の距離 |
| back_length | 肩中点 → 腰中点の距離 |
| back_angle | 肩中点 → 腰中点のベクトルと鉛直軸の角度 |
| head_angle | 鼻 (0) → 肩中点のベクトル角度 |
| shoulder_angle | 左肩 (11) と右肩 (12) の水平からの角度 |
| hip_angle | 左腰 (23) と右腰 (24) の水平からの角度 |
| neck_base, neck_width, head_width | 顔ランドマーク (Face Mesh 併用) |
| posture | 背中角度から "N"/"U"/"W"/"丸" を判定 |
| neck_hip, hip_knee | 既存距離計算 |
| right/left_shoulder_width | 肩 → 耳の水平距離 |
| right/left_ear_shoulder | 耳 → 肩の垂直距離 |

⚠️ **精度検証が必要**: Apple Vision と MediaPipe では座標系・スケールが異なるため、既存の閾値（例: [recommends/serializers.py:811](ソースコード/recommends/serializers.py#L811) の `shoulder_angle >= 27`）と合わない可能性。**キャリブレーション必須**。

### バックエンド: Netlify Functions (TypeScript)

**ファイル構成**
```
netlify/functions/
├── recommends.ts     # POST /api/recommends
├── orders.ts         # POST /api/orders
├── measure.ts        # POST /api/measure
├── notify.ts         # 内部: メール/LINE 送信
└── lib/
    ├── supabase.ts   # DB クライアント
    ├── recommend.ts  # レコメンドアルゴリズム (Python から移植)
    ├── pillow.ts     # 枕計算ロジック
    ├── mat.ts        # マットレス計算ロジック
    └── validation.ts # Zod スキーマ
```

**Python → TypeScript 移植対象**

| 元ファイル | 元コード | 移植先 |
|---|---|---|
| [recommends/serializers.py:537-617](ソースコード/recommends/serializers.py#L537-L617) | `generate_recommend_support_pad_place_by_data` | `lib/recommend.ts` |
| [recommends/serializers.py:620-668](ソースコード/recommends/serializers.py#L620-L668) | `generate_recommend_mat_by_data` | `lib/mat.ts` |
| [recommends/serializers.py:671-722](ソースコード/recommends/serializers.py#L671-L722) | `decimal_proc`, `calc_pillow_height_by_bmi` | `lib/pillow.ts` |
| [recommends/serializers.py:724-918](ソースコード/recommends/serializers.py#L724-L918) | `calc_pillow_height` | `lib/pillow.ts` |
| [recommends/serializers.py:921-1056](ソースコード/recommends/serializers.py#L921-L1056) | 単位変換・枕レコメンド生成 | `lib/pillow.ts` |

⚠️ **Decimal 型の扱い**: Python の `Decimal` ROUND_HALF_UP は、JavaScript の `Number` では再現できない。`decimal.js` ライブラリを使う必要あり。

### データベース: Supabase

**選定理由**
- PostgreSQL 完全互換（既存 [sql/](ソースコード/sql/) のスキーマ流用可能）
- 無料枠で月間500MB DB + 1GB ストレージ
- 管理画面（Supabase Studio）が Django Admin の代替になる
- Row Level Security (RLS) で Netlify Functions からの安全なアクセス

**スキーマ変換**
- Django の `UUIDField` → PostgreSQL `uuid`
- `models.ForeignKey` → `REFERENCES`
- `auto_now_add` → `DEFAULT NOW()`
- [sql/init.sql](ソースコード/sql/) を直接 Supabase に流し込めば移行可能（多少の手直しあり）

**改修 #3 で決定した変更も合わせて適用**
- `Measure.user` / `Order.user` 等の FK 削除（または null=True）
- `PillowRec` / `MatRec` に `claim_token` / `is_claimed` / `claimed_at` 追加
- `NotificationSetting` テーブル新設

### 管理画面: Supabase Studio + カスタム画面

**Django Admin の代替**
- **Supabase Studio**（標準付属）: テーブル CRUD、SQL エディタ、ユーザー管理
- **Retool** or **NocoDB**（オプション）: より Django Admin に近い UI が必要な場合
- **Next.js 管理ページ**（自作）: 注文一覧、計測履歴、通知設定の編集画面

**superuser ログインは?**
- Supabase Auth を利用（email + password）
- Netlify Functions に `supabase.auth.getUser()` で権限チェック
- 管理画面は `/admin/*` パスで保護

### 画像ストレージ: Netlify Blobs

**選定理由**
- Netlify 標準、追加設定不要
- Functions から直接読み書き
- 自動で CDN 経由配信

**代替**: Supabase Storage（DB と同じプロジェクトで管理したい場合）

### メール送信: Resend

**選定理由**
- 月間 3,000通 まで無料
- API がシンプル、TypeScript SDK 公式提供
- Netlify Functions との親和性が高い

**代替**: SendGrid、Amazon SES（ただし Netlify からの利用は非推奨）

### LINE 通知: LINE Messaging API (Push Message)

- 改修 #3 Q2 で決定済み
- 無料枠: 月間 200通（2024年改定）
- `lineChannelAccessToken` + `toUserId` を Supabase の `notification_settings` テーブルに保存
- Netlify Functions から `POST https://api.line.me/v2/bot/message/push`

## ⚠️ リスクと対策

### 🔴 高リスク

#### 1. MediaPipe と Apple Vision の計測値ズレ
**影響**: 既存のレコメンドアルゴリズム（閾値ベース）が合わなくなり、誤った推奨が出る

**対策**:
- Phase 0 でキャリブレーション必須
- 同じ人を両方で計測して補正係数を決める
- 場合によってはアルゴリズムの閾値を再調整（[recommends/serializers.py](ソースコード/recommends/serializers.py) の全条件式を見直し）

#### 2. Netlify Functions の 10秒タイムアウト
**影響**: レコメンド計算が10秒超えると失敗

**対策**:
- アルゴリズムは実質数十ms で終わるはずなので問題なし
- ただし DB アクセス込みで測定必須
- 超えるなら Netlify Background Functions（最大15分）に切替

#### 3. スマホブラウザのカメラ権限・パフォーマンス
**影響**: iOS Safari の制約（HTTPS 必須、ユーザーアクション必須）で動作しない可能性

**対策**:
- 必ず HTTPS（Netlify は標準対応）
- カメラ起動は明示的なボタンクリック経由
- 低スペック端末でも動くよう MediaPipe の `modelComplexity: 0` (Lite) から試す

### 🟡 中リスク

#### 4. Python → TypeScript 移植時のバグ混入
**影響**: 計算結果が微妙にズレる

**対策**:
- 既存 Python コードに対してユニットテストを作成（入力→期待出力のペア）
- 同じテストケースを TypeScript 側でも実行し一致を確認
- `decimal.js` で Python `Decimal` を忠実に再現

#### 5. Supabase 無料枠超過
**影響**: サービス停止

**対策**:
- 画像は Netlify Blobs に逃がす（Supabase はメタデータのみ）
- 古い計測データを定期削除するバッチ
- 有料プランへの移行タイミングを事前に決める

#### 6. LINE Messaging API の無料枠（200通/月）
**影響**: 超過分は課金

**対策**:
- メール通知と併用し、LINE は重要通知のみ
- 月間配信数を管理画面に表示

### 🟢 低リスク

#### 7. iOS 既存アプリとの並行運用
**影響**: 旧 Django バックエンドを一時的に維持する必要
**対策**: iOS アプリを廃止するか、Supabase に直接書き込む形に iOS を改修するかを決める
